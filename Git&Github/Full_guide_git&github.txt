** 💻 TERMINAL SETUP FOR GIT 💻
REASON : Tell Git your author name and email so commits are saved correctly  ✍️ 

-- Open terminal in VS Code and type:
(i) git config --global user.name " put user name here " --click enter
(ii) git config --global user.email " put email of github "  --click enter

--- only for checking (if the email shows in the terminal, then everything is set correctly) ✅
(i) git config --global --list



         📁📌 REPOSITORY INITIALISE IN VS CODE AND REMOTE ADDITION 📌📁

  ** Repository initialisation [ Method - 1 ]

** OPEN TERMINAL IN VS CODE TYPE TEXT ..
(i) git clone <repository-url>  [Do not include quotes; paste the URL and press enter]

** 🔗 Remote Addition [ Method - 1 ]

-- click on three dot choose ' remote '-- click on ' add remote '
-- then provide URL for 'HTTPS' for repository then click enter
-- then provide remote name ' origin ' [ "origin" general name choose as you wish]

📌 NOTES: WHY WE NEED REMOTE ADDITION ?
-- for push our code in github [This is global repository]

** 🔧 Remote Addition [ Method - 2 ]

.. USING COMMAND IN TERMINAL

-- git remote add <remote-name> <repository-url>
   [press enter; do not include quotes; use a space between the name and URL]

NOTES :

How to check remote ?
-- git remote -v
How to remove remote ?
-- git remote remove <remote-name>  ❌ [only put the remote name; do not include quotes]


** Repository initialisation [Method - 2 ]

(i) Make a folder and open in vs code [if you have already a folder don't need new folder]
.. open vs code terminal ..
(ii) git init  [then press enter; this initializes the local repository in VS Code]  📁

** Remote Addition

-- click on three dot choose ' remote '-- click on ' add remote '
-- then provide URL for 'HTTPS' for repository then click enter
-- then provide remote name ' origin ' [ "origin" general name choose as you wish]

📌 NOTES : If you add a different repository URL to the remote, then your push will go to that repository, not the initial local one.



                    🚀🚀🚀 CODE PUSH UP ON GITHUB 🚀🚀🚀

📌 NOTES: If you initialize a repository in VS Code and also add the remote URL and remote name, then you'll be able to push code.

*** METHOD - 1 ***

  -- ✅ Stage all files you want to push on GitHub
  -- 📝 Add a commit message
  -- 🔹 Click the "tick indication arrow" just above the commit, then wait a few seconds
  -- ▶️ Click the three dots on the right side, choose "Push or Pull", then click "Push" and wait a few seconds

      CONGRATULATIONS! YOUR CODE HAS BEEN SUCCESSFULLY PUSHED TO GITHUB REPOSITORY

*** METHOD - 2 ***
...Using terminal command

(i) git add .  ✅ [click enter - stages all changes]
(ii) git commit -m "put some message for commit"  📝
NOTES : -m mean adding some message in commit
FINAL COMMAND
(iii) git push <remote-name> <branch-name>  ▶️ [click enter - branch is usually "main" and VS Code shows your current branch]

NOTES : USE THIS WHEN YOU WANT TO AVOID A SEPARATE "git add ." COMMAND

  (i) git commit -a -m "put some message for commit"
  [click enter - -a stages all changed files and commits them]

        (ii) git push <remote-name> <branch-name>
   [click enter - branch is usually "main" and VS Code shows your current branch]
